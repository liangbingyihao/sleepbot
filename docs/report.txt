# 早睡锁屏App - 睡眠报告模块 PRD（完整版整合需求）
## 文档概述
1. 模块名称：睡眠报告（日报 / 周报 / 月报）
2. 渲染方案：App内本地WebView加载HTML模板；支持生成线上H5分享链接发给好友
3. 核心改动：废弃固定23:00-07:00统计时段，全部基于**用户自定义睡眠锁屏时段**计算；增加不健康睡眠时段校验与健康提示；适配无问卷基线计算挽回熬夜时长；统一分享规则
4. 分享约束：单次仅可分享当前Tab单份报告（日报/周报/月报三选一），不支持一次性打包多份分享

# 一、全局通用基础规则
## 1. 睡眠统计时段规则
1. 废弃固定23:00~07:00，统计区间以用户手动设置、同步后端的**自定义睡眠起止时间**为准；
2. 所有指标（有效锁屏时长、解锁次数、挽回时长、日历色块）仅在自定义时段内统计；
3. 报告页面展示的「锁屏时段」文案动态读取用户配置，不写死固定时间。

## 2. 自定义时段健康校验机制
### 健康标准
- 推荐健康区间：7h ~ 9h
- 健康最低底线：≥6h
- 不健康判定：自定义时段总时长 ＜6h
- 睡眠过长温和提示：自定义时段总时长 ＞12h

### 设置页保存拦截逻辑
1. 用户选好起止时间，实时计算总时长：
   - 时长＜6h：弹出警示弹窗
     文案：当前设置睡眠时长仅X小时，低于健康最低6小时标准，长期睡眠不足会影响精神与免疫力，建议调整至7-9小时。
     操作按钮：【重新调整】 / 【仍坚持使用】
     用户点击「仍坚持使用」才可保存，后端标记字段 `sleepIsUnhealthy = true`；
   - 时长＞12h：轻提示弹窗，无强制拦截，确认即可保存；
   - 6h ≤ 时长 ≤12h：无弹窗，直接保存。
2. 后端存储标记 `sleepIsUnhealthy`，全报告、分享H5共用该标识。
3. 用户修改时段至≥6h，标识自动重置为false，所有健康提示不再展示。

### 长期软性引导
1. 连续7天使用不健康睡眠配置：打开睡眠报告页弹出轻引导弹窗
   文案：你长期设置睡眠时长较短，是否前往设置调整更长睡眠区间？
   按钮：稍后再说 / 去调整
2. 每月1日生成月报时，若整月均为不健康时段，触发一次同款引导弹窗。

## 3. 每日睡眠三档判定标准（日历全局统一）
以用户自定义时段总时长 `totalCustomTime` 为基准，沿用原比例换算：
- 🟢 绿色打卡成功：有效锁屏时长 ＞ totalCustomTime × 0.875
- 🟡 黄色轻度熬夜：totalCustomTime × 0.625 ≤ 有效锁屏时长 ≤ totalCustomTime × 0.875
- 🔴 红色熬夜失败：有效锁屏时长 ＜ totalCustomTime × 0.625
补充备注：若用户标记为不健康作息，所有色块旁增加小字「判定仅基于你自定义时段，不代表达到健康睡眠标准」。

## 4. 三大核心统计指标
### 指标1：有效锁屏时长
自定义睡眠时段内，手机保持锁定、未点亮屏幕的累计时长；代表自律不玩手机时长。

### 指标2：夜间解锁次数
自定义睡眠时段内点亮/解锁手机总次数，次数越低自控力越强。

### 指标3：挽回熬夜时长（无用户问卷，纯行为基线算法）
1. 新用户有效睡眠记录＜3晚：隐藏该模块，占位文案：积累3晚睡眠数据后，为你生成自律提升分析；
2. 有效记录≥3晚：取最早3晚平均玩手机时长作为个人熬夜基线；
3. 单晚挽回时长 = 基线玩手机时长 − 当晚自定义时段内实际玩手机时长；
4. 计算结果＜0：隐藏数字，替换提示文案「今晚解锁次数略有增加，明天继续稳住作息」；
5. 周期汇总（周/月）：周期内每日有效挽回时长累加；
6. 不健康作息用户专属文案：对比初期作息少玩手机XX小时，但整体睡眠时长不足，建议延长睡眠时段。

## 5. 时长展示规范
1. 单日日报：人性化格式 XX小时XX分钟；
2. 周/月汇总：同时展示总分钟数 + XX小时XX分钟两种格式。

## 6. 环比对比逻辑
1. 日报：对比用户自身初期基线；
2. 周报：环比上周同期数据；
3. 月报：环比上月同期数据。

## 7. 历史数据隔离规则
用户中途修改自定义睡眠时段：
- 修改**当日起**启用新时段统计规则；
- 历史旧数据沿用修改前的旧时段、旧判定标准，不回溯重算历史记录。

## 8. 全局异常兜底
1. 当日无锁屏、解锁记录：当日不计入任何统计，日历格子空白无颜色；
2. 挽回时长负数仅温和提示，不展示负面数字；
3. 连续打卡中断仅分割月历区块，不清空历史最高连续打卡天数；
4. 用户超90天无睡眠记录：清空历史基线，重新积累3晚数据生成新基线；
5. 全报告文案正向鼓励导向，无打击式负面描述。

# 二、报告入口与页面整体架构
## 1. 首页入口
首页功能卡片「睡眠报告」；
每日/每周/每月生成新报告时，卡片右上角点亮小红点提示。
点击进入二级聚合报告页面。

## 2. 二级睡眠报告聚合页
### 页面布局
1. 顶部导航栏：左侧标题「睡眠报告」｜中间「历史记录」按钮｜右上角「分享」按钮
2. 横向可滑动Tab栏（固定顺序）：日报 / 周报 / 月报，默认选中【日报】
3. 主体区域：单例WebView，加载本地缓存HTML模板，切换Tab仅调用JS方法`renderReport()`重渲染数据，不重载页面；
4. 底部无额外固定按钮。

### 历史记录弹窗功能
点击「历史记录」弹出筛选弹窗，分三类查看历史报告：
1. 历史每日：日历选择任意日期，打开对应日报；
2. 历史每周：选择周区间，打开对应周报；
3. 历史每月：选择月份，打开对应月报。

# 六、日历组件统一规范
1. 周简易日历：周报页面展示本周7天色块，仅可视化，无点击交互；
2. 完整月历：月报页面展示当月全部日期格子；
   - 三色填充规则全局统一；
   - 自动高亮连续打卡区块；
   - 点击单日弹窗展示对应完整日报（弹窗同步展示用户自定义锁屏时段、健康提示）。

# 七、分享功能完整需求
## 1. 基础分享规则
1. 分享按钮常驻页面右上角，切换任意Tab均可点击；
2. **单次分享仅支持当前Tab单份报告**：切日报分享日报、切周报分享周报、切月报分享月报；
3. 不支持一键打包日报+周报+月报合并分享，如需多份发给好友，需切换Tab多次分享；
4. 分享弹窗底部小字提示：每次仅可分享当前页面单份报告，如需分享其他周期请切换标签后再次分享。

## 2. 分享弹窗操作选项
1. 复制分享链接；
2. 唤起微信/QQ原生分享面板；
3. 生成报告图片海报，保存至本地相册。

## 3. 分享链接与线上H5页面规则
1. 链接携带唯一参数区分用户、周期、报告类型：
   - 日报：`?type=day&date=2026-06-22`
   - 周报：`?type=week&week=2026w25`
   - 月报：`?type=month&month=2026-06`
2. 后端生成临时有效链接（默认有效期30天，过期提示报告已失效）；
3. 线上H5使用与App本地完全同一套HTML模板，自动读取用户自定义睡眠时段、`sleepIsUnhealthy`标记；
4. H5页面只读，移除Tab切换、历史记录等App内部功能；
5. 不健康作息分享页同步展示黄色健康警示提示；
6. 页面底部增加回流引导文案：想要记录自己的睡眠作息？下载早睡锁屏App；
7. 数据安全：链接携带后端签名防篡改，好友仅可查看当前分享周期数据，无法查看用户其他历史睡眠记录。

## 4. 模板升级兼容说明
1. App内置HTML作为离线兜底模板；同时服务端部署同款HTML用于分享H5；
2. 采用「内置兜底 + 云端热更新」方案更新模板，修改后两端同步生效；
3. JS数据结构向下兼容，新增字段做判空处理，新旧模板均可正常渲染。

# 八、HTML模板渲染逻辑
1. App本地WebView环境：通过JSBridge接收原生传入JSON数据，调用 `renderReport(data, type)` 渲染；
2. 线上分享H5环境：自动解析URL参数，AJAX请求后端拉取报告数据渲染；
3. 页面自动区分运行环境，两套渲染逻辑互不冲突；
4. 移动端强制视口适配，禁止双指缩放，尺寸使用vw/rem自适应；
5. 三色状态通过CSS类动态切换，空态页面单独占位样式。
